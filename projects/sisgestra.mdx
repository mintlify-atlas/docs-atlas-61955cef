## Overview

SisGestra (Sistema de Gestión de Transporte - Transport Management System) is a comprehensive web application designed for inventory and personnel management. The system was developed to digitize and optimize logistical processes in transport operations, featuring an intuitive interface, intelligent alert systems, and automated reporting via email.

The platform addresses real-world operational challenges by streamlining resource tracking, shift management, and communication workflows for teams managing vehicles, equipment, and personnel.

    ```python
    # Intelligent alert system implementation
    
    from django.db.models.signals import post_save, pre_save
    from django.dispatch import receiver
    
    @receiver(pre_save, sender=InventoryItem)
    def check_inventory_threshold(sender, instance, **kwargs):
        """Alert when inventory falls below threshold"""
        if instance.quantity < instance.minimum_threshold:
            Alert.objects.create(
                alert_type='LOW_INVENTORY',
                severity='HIGH',
                message=f"{instance.name} is below minimum threshold",
                related_item=instance
            )
            send_alert_email(
                subject=f"Low Inventory Alert: {instance.name}",
                recipients=['inventory@company.com'],
                context={'item': instance}
            )
    
    @receiver(post_save, sender=Shift)
    def check_shift_conflicts(sender, instance, created, **kwargs):
        """Detect scheduling conflicts"""
        if created:
            conflicts = Shift.objects.filter(
                employee=instance.employee,
                shift_date=instance.shift_date
            ).exclude(id=instance.id)
            
            if conflicts.exists():
                Alert.objects.create(
                    alert_type='SHIFT_CONFLICT',
                    severity='MEDIUM',
                    message=f"Shift conflict detected for {instance.employee}"
                )
    ```

    ```python
    # Automated email reporting system
    
    from celery import shared_task
    from django.core.mail import EmailMultiAlternatives
    from django.template.loader import render_to_string
    
    @shared_task
    def send_daily_report():
        """Generate and send daily operations report"""
        # Gather statistics
        stats = {
            'active_vehicles': Vehicle.objects.filter(status='active').count(),
            'on_duty_staff': Shift.objects.filter(
                shift_date=date.today(),
                start_time__lte=now().time(),
                end_time__gte=now().time()
            ).count(),
            'low_inventory_items': InventoryItem.objects.filter(
                quantity__lt=F('minimum_threshold')
            ).count(),
            'maintenance_due': Vehicle.objects.filter(
                next_maintenance__lte=date.today() + timedelta(days=7)
            ).count(),
        }
        
        # Render email template
        html_content = render_to_string('reports/daily_report.html', stats)
        text_content = render_to_string('reports/daily_report.txt', stats)
        
        # Send email
        email = EmailMultiAlternatives(
            subject=f"Daily Operations Report - {date.today()}",
            body=text_content,
            from_email='reports@sisgestra.com',
            to=['management@company.com']
        )
        email.attach_alternative(html_content, "text/html")
        email.send()
    ```

### Key Design Decisions

<Steps>
  <Step title="Requirements Analysis">
    Conducted thorough analysis of transport logistics workflows to identify pain points in manual processes, leading to targeted digitization of critical operations.
  </Step>
  
  <Step title="Database Design">
    Created normalized database schema to eliminate data redundancy while maintaining referential integrity across vehicles, personnel, and inventory.
  </Step>
  
  <Step title="Alert Architecture">
    Implemented Django signals for real-time event detection and response, ensuring immediate notification of critical situations.
  </Step>
  
  <Step title="Reporting Automation">
    Designed scheduled task system using Celery for background job processing, enabling reliable report generation without impacting user experience.
  </Step>
</Steps>
