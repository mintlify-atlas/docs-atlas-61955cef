## Overview

Django Suites Hotel is a comprehensive web platform designed for hotel reservation, administration, and room service management. This project showcases a full-featured hotel management system with a focus on user experience and operational efficiency.

The application provides both guest-facing and administrative interfaces, enabling seamless booking experiences while giving hotel staff powerful tools to manage reservations, room availability, and service requests.

## Technical Implementation

### Database Architecture

The application uses a carefully designed relational database schema with the following core models:

<Tabs>
  <Tab title="Models Overview">
    ```python
    # Core Django models
    
    class Room(models.Model):
        """Hotel room with all details and pricing"""
        room_number = models.CharField(max_length=10, unique=True)
        room_type = models.CharField(max_length=50)
        capacity = models.IntegerField()
        price_per_night = models.DecimalField(max_digits=8, decimal_places=2)
        description = models.TextField()
        amenities = models.JSONField()
        status = models.CharField(max_length=20)
        
    class Reservation(models.Model):
        """Guest booking records"""
        guest = models.ForeignKey(User, on_delete=models.CASCADE)
        room = models.ForeignKey(Room, on_delete=models.CASCADE)
        check_in_date = models.DateField()
        check_out_date = models.DateField()
        total_price = models.DecimalField(max_digits=10, decimal_places=2)
        status = models.CharField(max_length=20)
        
    class RoomServiceOrder(models.Model):
        """Room service requests from guests"""
        reservation = models.ForeignKey(Reservation, on_delete=models.CASCADE)
        order_items = models.JSONField()
        order_time = models.DateTimeField(auto_now_add=True)
        status = models.CharField(max_length=20)
        total_amount = models.DecimalField(max_digits=8, decimal_places=2)
    ```
  </Tab>
  
  <Tab title="Relationships">
    The database implements several key relationships:
    
    - **User → Reservation**: One-to-many (guests can have multiple reservations)
    - **Room → Reservation**: One-to-many (rooms can be booked multiple times)
    - **Reservation → RoomServiceOrder**: One-to-many (orders tied to specific reservations)
    - **Room → Images**: One-to-many (multiple photos per room)
    
    These relationships ensure data integrity and enable complex queries like:
    - Finding all reservations for a specific date range
    - Calculating room occupancy rates
    - Tracking guest spending on services
  </Tab>
  
  <Tab title="Queries">
    ```python
    # Example: Check room availability
    def check_availability(room_id, check_in, check_out):
        overlapping = Reservation.objects.filter(
            room_id=room_id,
            status__in=['confirmed', 'checked_in'],
            check_out_date__gt=check_in,
            check_in_date__lt=check_out
        ).exists()
        return not overlapping
        
    # Example: Calculate total revenue
    def get_monthly_revenue(year, month):
        return Reservation.objects.filter(
            check_in_date__year=year,
            check_in_date__month=month,
            status='completed'
        ).aggregate(total=Sum('total_price'))['total']
    ```
  </Tab>
</Tabs>

### Key Features Implementation

<Steps>
  <Step title="User Authentication">
    Implements Django's built-in authentication system with custom user profiles, role-based access control (guest vs. staff), and secure password handling with session management.
  </Step>
  
  <Step title="Booking Engine">
    Real-time availability checking using PostgreSQL date range queries, automatic price calculation based on nights and room type, and conflict prevention through database constraints.
  </Step>
  
  <Step title="Room Service Integration">
    JSON-based order storage for flexible menu items, real-time order status updates, and kitchen notification system using Django signals.
  </Step>
  
  <Step title="Admin Dashboard">
    Custom Django admin interface with advanced filtering, bulk operations for managing multiple reservations, and visual analytics using Chart.js integration.
  </Step>
</Steps>

## Development Highlights

<Accordion title="Challenges & Solutions">
  **Challenge:** Managing concurrent booking requests to prevent double-booking
  
  **Solution:** Implemented database-level transactions and row-level locking to ensure only one reservation can be created for a room during overlapping dates.
  
  ---
  
  **Challenge:** Calculating dynamic pricing based on season, room type, and length of stay
  
  **Solution:** Created a flexible pricing engine using Django models with date-based rules and automatic calculation in the booking view.
  
  ---
  
  **Challenge:** Real-time updates for room service orders
  
  **Solution:** Used Django signals to trigger notifications and status updates when orders are modified by kitchen staff.
</Accordion>
